반환 o

출력문 괄호안에 매개변수로 호출문이 사용 -> 호출문의 대상은 반환값이 있어야한다.

반환값이 없는 호출문 -> 자기 혼자 호출해야함 (= 연산자 오른쪽에 안옴)

반환값이 있는 메소드는 메소드 안에 하나 이상의 리턴문이 와야함

void 에 리턴문 쓸 수 있음 : 제어를 반납한다           

한 클래스 내에서 하나만 존재가능(이름겹치면 x)

메소드 오버로딩 : 전제조건   - *매개변수 조건,타입,배열상태가 달라야함(하나만 성립되어도 됨)
			- 하나의 클래스 안에서 일어나는 현상
			- 이름이 똑같아야함(반환값은 상관없음)
			- 변수명이 다르면 오버로딩 x
			- 타입,순서,개수가 달라야 오버로딩


오버로딩 사용이유 : 다양한데이터를 이용해서 연산하기위해. 하나의 메소드로 수많은 타입의 데이터를 처리하기 위해서 사용함


메소드 - 선언부(프로토타입)(구조?형태?) : 접근지정자 제한자 반환타입 | 보이드 메소드명(매개변수){}


선언부		접근지정자(4가지)public protected private default
		제한자 static final abstract

선언부 : 메소드의 기능을 제한할 때 사용 static 생성되어지는 시점을 지정함 nwe연산자 - 객체 생성되면 이 메소드도 같이 생성
	그러나 static이 붙은 놈은 객체가 생성되기 훨씬 전부터(이클립스가 메모리에 로딩될 때 부터) 이미 만들어져있음
	메소드와 변수에 붙을 수 있음
	과거가 미래데이터를 불러다 쓸 수 없다. 
					★(static인 메소드가 static 바깥에 있는 변수나 메소드를 부르고자 했을 때에는
					불림받는 놈도 static 이어야함)
					(static이 붙은 놈은 static이 안붙은놈 불러다 쓸수 없음)
	static : 정적
	변수에 static이 붙으면 생성되어지는 것도 저 과거에 생성됨
	 힙이 아닌 맨위 메소드에리아에 생성되어짐. 그리고 딱 하나의 공간만 배정되어 생성됨. 
	초기화도 그때 딱 한번 발생.
	모든 클래스들이 공통으로 사용
		멤버변수는 객체가 생성될 때 마다 초기화가 일어남

final 변수,메소드,클래스 다 붙을 수 있음
      final + 변수 : 상수
      final + 메소드 : 오버라이드(재정의 금지)
      final + 클래스 : 불인클래서(자식을 가질수 없음. 종단클래스. 상속불가능)

abstract : 추상클래스가 되어질 수 있는 메소드를 만들 때 사용(추상메소드를 하나 이상 가지고 있으면 클래스도 추상클래스가 되어야함)
		추상메소드 - 바디가 없음(이름과 괄호안에 있는 매개변수만 있음). 실제 코드가 존재하지 않는 것을 추상메소드 라고 한다
		추상메소드 앞에는 반환타입명 앞에 abstract이 붙어야함
		추상클래스는 객체로 만들 수 없음 걔를 상속받은 자식클래스가 객체가 됨


생성자메소드도 오버로딩의 대상이다.

생성자메소드의 역할 : 멤버변수 초기화

기본생성자 : 매개변수가 없는 생성자

★사용자가 클래스를 만들면서 기본생성자를 하나도 생성하지 않았을때 컴파일러가 자동으로 하나 만들어서 넣어준다.
 단, 사용자가 하나라도 만든다면 자동생성되지않는다

생성자메소드 = 클래스에 존재하는 멤버필드를초기화하기위함

this != this()

this = 자신의 클래스주소를 보관하는 참조형변수
	매개변수의 이름과 멤버변수의 이름이 똑같을 때 메소드내에서 지역변수가 아닌 멤버필드를 지정할 때 this사용
this() = 쓰이는 곳이 딱 한군데로 제한. 자신의 생성자 메소드에서만 사용
	생성자 메소드의 첫 줄에 사용되어 다른 생성자 메소드 호출(다른메소드(다른생성자메소드)를 호출하기위해사용)

2) 트랜잭션
 - 데이터베이스에서 하나의 논리적 기능을 수행하기 위한 일련의 연산집합
 
 트랜잭션의 상태 
- 완료되면 commit 
- 비정상적종료되면 ROLLBACK

릴레이션 - 테이블 : 정보를 구분하여 저장하는 기본단위

튜플 - 레코드 : 개체에 속한 구성원들 개개인의 정보를 표현

속성 - 필드(칼럼) : 릴레이션에서 관리하는 구체적인 정보항목

도메인 - 속성의 데이터 형 : 각각의 속성들이 가질 수 있는 값들의 집합

키종류를 이야기하시오 
 - 기본키, 대레키, 후보키, 복합키, 왜래키
후보키 -> 각 튜플을 구별하는데 기준이 되는 컬럼의 집합
기본키 -> 기준으로 사용할 키
대체키 -> 후보키 중 기본키로 선택되지 않는 나머지 키
복합키 -> 두 개 이상의 컬럼이 합쳐져야 후보가 역할을 함
외래키 -> 테이블과 테이블간의 연결성이 있다
