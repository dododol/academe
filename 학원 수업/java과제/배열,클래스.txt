클래스 = 데이터 타입

객체지향언어의 특징
상속, 다형성, 정보의 은닉성
->한 단어로 표현하면 재사용

클래스는 수동적이다. 움직여줘야하는데 그러기 위해 필요한 것이 객체화이다.
클래스는 설계도면이다. 도면은 움직이지 않음
클래스를 이용해 만들어진 물건이 객체이다.
-> 이 과정을 인스턴스, 인스턴스화 라고 한다

클래스 포함요소
멤버
1) 변수(속성) -> 클래스의 특성
2) 메소드(행동) 


대문자로 시작하면 클래스

private,생성자메소드 상속 x

부모클래스가 먼저 생성되어야 자식클래스에서 상속할수있다

클래스를 사용하려면 접근지정자를 써야한다. 
                동일클래스             동일패키지         다른패키지내의 자식클래스	다른패키지

하나의 파일에는 public이 하나만 붙을 수 있다.

접근지정자	제한자	class	클래스명

제한자 - abstract : 추상클래스를 만들어줌. 
	final : 변수앞에 붙으면 변수를 상수로 바꿈. 클래스앞에 final이 붙으면 상속이 금지됨.
